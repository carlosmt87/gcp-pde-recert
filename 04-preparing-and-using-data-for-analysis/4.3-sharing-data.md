## Key Concepts

> _Fill in your own notes here as you study._

- **Analytics Hub**: GCP's managed data exchange platform built on BigQuery. Enables
  organizations to publish, discover, and subscribe to data products (called *listings*)
  across organizational boundaries — internally, between companies, or publicly.

- **Data Exchange**: An Analytics Hub exchange is a curated marketplace of datasets.
  Publishers (data providers) create exchanges and add listings. Subscribers (data
  consumers) browse and subscribe to listings.

- **Listings**: A listing is a pointer to a BigQuery dataset that is shared. When a
  subscriber subscribes to a listing, a *linked dataset* is created in their project
  — a read-only, zero-copy reference to the publisher's data. No data duplication.

- **Zero-copy sharing**: The linked dataset in the subscriber's project does not
  duplicate the data. The subscriber's BigQuery jobs query the publisher's storage
  directly, but in the subscriber's billing context (subscriber pays for query costs).

- **Cross-organization sharing**: Analytics Hub supports sharing between different
  GCP organizations, enabling data marketplace scenarios (e.g., a data vendor sharing
  weather data with multiple customers).

- **Comparison to other sharing methods**:
  - *Authorized views*: Share specific query results within a project/org; no versioning
  - *IAM on datasets*: Grant cross-project access; requires managing individual IAM bindings
  - *Analytics Hub*: Managed, versioned, discoverable, with governance — preferred for
    formal data product sharing at scale
  - *BigQuery Data Transfer Service*: Copies data between projects; introduces duplication

- **Governed data sharing**: Analytics Hub integrates with VPC Service Controls and
  IAM, enabling data providers to maintain control over who can subscribe and ensuring
  data remains within approved perimeters.

## Practice Questions

### Question 1

A financial data company sells curated market data to 200 enterprise customers. Each
customer uses GCP and queries the data with BigQuery. The company wants customers to
access the data in their own BigQuery projects without the company incurring query costs
on behalf of customers, and without duplicating data across 200 projects. Which solution
best meets these requirements?

**A)** Copy the BigQuery dataset to each customer's project using the BigQuery Data Transfer Service
**B)** Grant each customer's service account `roles/bigquery.dataViewer` on the source dataset
**C)** Publish the dataset as a listing in Analytics Hub; customers subscribe and get linked datasets in their own projects
**D)** Create authorized views in each customer's project pointing to the source dataset

**Answer: C**

> Analytics Hub listings create zero-copy linked datasets in each subscriber's project.
> Customers pay for their own query costs. No data duplication (vs. A). No per-customer
> IAM management (vs. B). Analytics Hub also provides a governed marketplace with
> discoverability. The financial company maintains control over subscription approvals.

### Question 2

An internal data platform team wants to share a curated BigQuery dataset with business
units across 5 different GCP projects within the same organization. The data platform
team should retain control over the data schema and content; business units should only
be able to query, not modify. The sharing mechanism should be discoverable so teams
can self-service subscribe. Which is the most appropriate approach?

**A)** Use IAM to grant `roles/bigquery.dataViewer` to each project's service account on the dataset
**B)** Create an Analytics Hub exchange, publish the dataset as a listing, and let business units subscribe to create linked datasets
**C)** Use BigQuery authorized views, one per business unit project
**D)** Replicate the dataset to each business unit's project nightly with Cloud Composer

**Answer: B**

> Analytics Hub provides a self-service, discoverable marketplace. Business units browse
> the exchange and subscribe independently. IAM (A) works but requires manual management
> and isn't self-service or discoverable. Authorized views (C) are not self-service;
> replication (D) duplicates data and introduces staleness.

### Question 3

A company shares sensitive financial data via Analytics Hub. They need to ensure that
subscribers can only access the linked dataset from within approved GCP projects and
cannot download the data to external environments. Which additional control should the
data engineer configure?

**A)** Set the Analytics Hub listing to "Private" mode with manual approval
**B)** Configure VPC Service Controls to include both the publisher's and subscribers' projects in the same service perimeter
**C)** Add a Cloud DLP inspection job on the subscriber's linked dataset
**D)** Enable BigQuery audit logging on the publisher's dataset

**Answer: B**

> VPC Service Controls around the publisher's project and approved subscriber projects
> prevent data exfiltration — even if a subscriber has a linked dataset, they can only
> query it from within the approved perimeter. Private mode (A) controls who subscribes
> but not where they query from. Audit logging (D) provides visibility but not prevention.
