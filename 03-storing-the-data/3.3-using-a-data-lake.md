## Key Concepts

> _Fill in your own notes here as you study._

- **Data lake architecture**: A centralized repository storing raw data at scale in
  its native format (structured, semi-structured, unstructured). On GCP, Cloud Storage
  is the foundation; Dataplex provides the governance layer.

- **Dataplex Lakes, Zones, and Assets**:
  - *Lake*: Top-level logical grouping (e.g., "enterprise-data-lake")
  - *Zone*: Organizational subdivision by data type or lifecycle stage
    - Raw zone: Unprocessed, unvalidated data
    - Curated zone: Validated, transformed, business-ready data
  - *Asset*: A Cloud Storage bucket or BigQuery dataset attached to a zone

- **Data discovery in Dataplex**: Automatic metadata extraction from Cloud Storage
  files — infers schema, detects file formats, creates Hive-compatible BigQuery external
  tables. Makes data searchable via Dataplex Catalog.

- **Access controls**: Dataplex IAM roles allow centralized permission management
  across all assets in a lake, rather than per-bucket Cloud Storage policies. Supports
  coarse-grained (lake/zone) and fine-grained (asset) access.

- **Cost controls in a data lake**:
  - Storage class lifecycle rules: Automatically move objects to Nearline, Coldline,
    or Archive based on last-access time
  - Object versioning: Enable only where needed (adds storage cost)
  - Requester pays: Shift egress/read costs to data consumers
  - BigQuery slot reservations: Prevent runaway query costs on lake data
  - Dataplex data tiering: Manage hot/warm/cold data placement

- **Data lake anti-patterns**: "Data swamps" arise without governance — enforce schemas,
  data quality rules, and access controls from day one.

| Service | Role |
|---------|------|
| **Dataplex** | Data lake management — zones, assets, discovery, quality, security |
| **Cloud Storage** | Physical storage for raw and curated data lake files |
| **Dataplex Catalog** | Metadata management and discovery across lake assets |
| **BigQuery** | Query engine for lake data; curated zone data often resides here |
| **BigLake** | Unified governance for Cloud Storage data queried via BigQuery |
| **Cloud Storage Lifecycle** | Automated cost-optimized storage class transitions |
| **IAM** | Access control for lake zones and assets |

## Practice Questions

### Question 1

A company uses Cloud Storage as their data lake. Data teams across the organization
complain that they cannot find relevant datasets, don't know what data is available, and
there is no consistent access control. The data engineer is tasked with improving this.
Which GCP solution addresses all three concerns?

**A)** Create a BigQuery dataset and copy all files from Cloud Storage into tables
**B)** Implement Dataplex with lakes and zones, enable automatic discovery, and manage IAM at the zone level
**C)** Set up a Cloud Storage bucket with a well-organized folder structure and a shared README
**D)** Deploy Dataproc with a Hive metastore and configure HDFS-compatible access

**Answer: B**

> Dataplex directly addresses all three issues: automatic discovery (metadata extraction
> and Catalog integration), access controls (centralized IAM across zones), and data
> organization (lakes/zones structure). Option A requires data movement; C and D don't
> solve the governance problem at scale.

### Question 2

A data lake stores raw log files that are accessed frequently for the first 30 days,
occasionally for up to 90 days, and rarely or never after that. The company wants to
minimize storage costs automatically without manual intervention. Which configuration
achieves this?

**A)** Manually move files between buckets based on age using a monthly script
**B)** Set the default storage class to Archive for the entire bucket
**C)** Configure Cloud Storage lifecycle rules: transition to Nearline at 30 days, Coldline at 90 days, Archive at 365 days
**D)** Delete all files older than 90 days with a lifecycle delete rule

**Answer: C**

> Lifecycle rules automatically transition objects between storage classes based on age,
> optimizing cost without manual intervention. Nearline (30-day min) suits occasional
> access; Coldline (90-day min) suits infrequent access. Setting Archive from the start
> (B) incurs retrieval fees for the frequent early access period.

### Question 3

A Dataplex lake contains a raw zone with JSON log files and a curated zone with
BigQuery datasets. A new data analyst team should be able to read curated zone data
but must NOT be able to access raw zone data containing PII. How should access be
configured?

**A)** Grant the analyst team `roles/storage.objectViewer` on all Cloud Storage buckets
**B)** Grant `roles/dataplex.dataReader` at the curated zone level; grant no permissions on the raw zone
**C)** Create a BigQuery authorized view of the curated zone data for the analyst team
**D)** Use VPC Service Controls to block raw zone access for the analyst team's IPs

**Answer: B**

> Dataplex zone-level IAM allows granting `dataplex.dataReader` specifically to the
> curated zone without affecting the raw zone. This provides clean, centralized access
> management without managing per-bucket permissions. Option C (authorized views) is
> valid for BigQuery data but doesn't cover Cloud Storage assets.
