# What's New in the Renewal Exam

This file highlights topics that are *specifically emphasized* in the **Renewal** exam
vs. the original PDE exam. If you've held the PDE cert before, these are the areas
most likely to have changed or been added.

---

## Services That Are New or Significantly Elevated

### Dataplex Catalog *(replaces standalone Data Catalog)*
The original PDE exam covered Cloud Data Catalog. The renewal reflects the migration
to **Dataplex Catalog**, which absorbs Data Catalog's functionality and adds:
- Unified governance across GCS, BigQuery, AlloyDB, and other sources
- Data lineage tracking (end-to-end from source to destination)
- Automatic metadata discovery (schema inference on GCS files)
- Custom aspects (flexible metadata schemas beyond tag templates)
- Integration with Dataplex data quality scanning

**What to know:** Dataplex Catalog *is* the current metadata/discovery tool. Anything
the old exam said about Data Catalog now applies to Dataplex Catalog.

---

### BigLake *(new in renewal)*
Not present in the original PDE exam.

**What to know:**
- Unified storage engine that extends BigQuery governance to Cloud Storage files
- Enables column-level security (Policy Tags) on external Parquet/ORC/Avro files
- Supports BigQuery SQL on GCS data without copying into native BQ storage
- The "lakehouse" pattern on GCP — key for Section 3 scenarios
- Contrast with plain external tables: BigLake enforces access control at the data level; external tables only at the BQ table level

---

### AlloyDB *(new in renewal)*
Not present in the original PDE exam.

**What to know:**
- Fully managed PostgreSQL-compatible database
- HTAP capable (handles both transactional and analytical workloads)
- Significantly faster than Cloud SQL for complex queries (~4x reads, ~2x writes)
- Supports `pgvector` extension for vector embeddings and similarity search
- Use when: high-performance PostgreSQL OLTP is needed, or when you need a relational
  store that also supports vector search for ML/RAG workloads

---

### Dataform *(elevated in renewal)*
Existed before but is now a first-class citizen in the exam.

**What to know:**
- SQL-based ELT framework integrated into BigQuery (replaces external dbt in many GCP architectures)
- Supports: assertions (data quality tests), incremental tables, Git-based version control
- Release configurations allow environment-specific variable injection (dev/staging/prod)
- Schedules workflow invocations via built-in cron or Composer/Workflows triggers
- Key distinction: Dataform is ELT-only (data must already be in or coming to BigQuery); it does not move data between systems

---

### BigQuery Editions *(replaced flat-rate pricing)*
The original exam covered on-demand and flat-rate pricing. Flat-rate has been replaced
by **Editions**.

**What to know:**
- Three tiers: Standard, Enterprise, Enterprise Plus
- All Editions use autoscaling slots (pay per slot-second used above baseline)
- Reservations + Assignments are still the mechanism for allocating capacity
- Idle slot sharing allows unused capacity to flow between reservations
- Autoscaling maximums cap cost; commitments (1yr/3yr) reduce per-slot cost
- See `gcp-service-comparisons.md` Section 8 for the full comparison table

---

### Analytics Hub *(elevated in renewal)*
More prominent than in the original exam.

**What to know:**
- Managed data exchange built on BigQuery
- Publishers create exchanges and listings; subscribers create zero-copy linked datasets
- Subscribers pay their own query costs (no data egress charges to publisher)
- Supports cross-organization sharing
- Integrates with VPC Service Controls for perimeter enforcement
- Key distinction from authorized views: Analytics Hub is discoverable, versioned,
  and self-service; authorized views require manual IAM management

---

### Generative AI and RAG *(entirely new in renewal)*
The original PDE exam had no generative AI content.

**What to know:**
- **Embeddings**: Dense vector representations of text/images capturing semantic meaning.
  Generated by Vertex AI Embeddings API (e.g., `text-embedding-005`).
- **Vector search**: Approximate nearest-neighbor search over embedding vectors.
  GCP options: Vertex AI Vector Search, BigQuery `VECTOR_SEARCH()`, AlloyDB pgvector.
- **RAG (Retrieval-Augmented Generation)**: Pattern for grounding LLM responses in
  proprietary data. Pipeline: extract → chunk → embed → index → retrieve → generate.
- **LLM-assisted query generation**: Using Gemini (via Vertex AI or BigQuery Gemini
  integration) to generate SQL from natural language. Reduces manual coding in pipelines.
- **Unstructured data prep**: Extracting text from PDFs/images (Document AI, Vision AI
  OCR) as a preprocessing step before embedding generation.

---

### Dataplex Data Quality *(elevated in renewal)*
**What to know:**
- Native data quality scanning within Dataplex (rules-based: not-null, regex, range,
  uniqueness, referential integrity)
- Scans can run on Cloud Storage files and BigQuery tables
- Results stored in BigQuery for querying and alerting
- Complement to Dataform assertions — Dataplex quality rules run on existing data
  without being tied to a transformation pipeline

## Topics Carried Over From Original PDE (Still Tested)

These are not new but remain core to the exam:

| Topic | Key Points |
|-------|-----------|
| Dataflow | Apache Beam, windowing, exactly-once, Flex Templates |
| Cloud Composer | Airflow DAGs, sensors, backfilling, BranchPythonOperator |
| Pub/Sub | At-least-once delivery, dead-letter topics, message ordering |
| Bigtable | Row key design, time-series, HBase compatibility |
| Cloud Spanner | Global OLTP, external consistency, interleaved tables |
| Cloud DLP | De-identification methods (tokenization, masking, bucketing) |
| IAM | Least privilege, service accounts, workload identity |
| Cloud Storage lifecycle | Storage classes, lifecycle rules for cost optimization |
| BigQuery partitioning + clustering | Partition pruning, clustering key selection |
| VPC Service Controls | Security perimeters, preventing data exfiltration |

## Renewal Exam Mindset Shifts

| Original PDE Focus | Renewal Focus |
|-------------------|--------------|
| Data Catalog | Dataplex Catalog |
| Flat-rate pricing | BigQuery Editions + Reservations |
| Basic external tables | BigLake (governed external tables) |
| ML model training pipelines | Feature Store, RAG, embeddings |
| Cloud SQL for relational | + AlloyDB for high-performance PostgreSQL |
| Data lineage as a concept | Dataplex lineage as a managed feature |
| Building governance manually | Dataplex as the governance control plane |
