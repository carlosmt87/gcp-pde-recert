# Section 2: Ingesting and Processing the Data

**Exam Weight: ~10%**

This section covers the end-to-end pipeline lifecycle: planning transformation logic,
building pipelines with the right tools, and deploying them in a repeatable, automated way.

## Subsections

| Subsection | Topic | File |
|------------|-------|------|
| 2.1 | [Planning the data pipelines](./2.1-planning-data-pipelines.md) | Data transformation and orchestration logic |

| 2.3 | [Deploying and operationalizing the pipelines](./2.3-deploying-and-operationalizing-pipelines.md) | Job automation, orchestration, CI/CD |

## Key Themes

- **Pipeline planning**: Matching the transformation approach to data volume, latency, and complexity
- **AI enrichment**: Using Vertex AI or pre-trained models to enrich data during processing
- **Orchestration**: Cloud Composer (Airflow) vs. Workflows for job scheduling
- **CI/CD for data**: Applying software engineering practices to pipeline deployment
