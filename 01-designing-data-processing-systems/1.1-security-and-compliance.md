## Key Concepts

> _Fill in your own notes here as you study._

- **Data sovereignty**: Data must reside and be processed within specific geographic
  boundaries to comply with local laws (e.g., GDPR in Europe, data residency laws in
  specific countries). GCP enforces this via region and multi-region configurations.

- **VPC Service Controls**: Create security perimeters around GCP resources to prevent
  data exfiltration, even by authorized users.

- **Customer-Managed Encryption Keys (CMEK)**: Use Cloud KMS to manage your own
  encryption keys for GCP services, meeting compliance requirements that prohibit
  Google-managed keys.

- **IAM best practices**: Principle of least privilege, use of service accounts,
  workload identity federation, and organization policies.

- **Regulatory frameworks**: HIPAA, PCI-DSS, SOC 2, ISO 27001 — understand which
  GCP services are compliant and how to architect solutions accordingly.

- **Audit logging**: Cloud Audit Logs capture Admin Activity, Data Access, and System
  Event logs for compliance evidence.

## Practice Questions

### Question 1

A financial services company processes customer transaction data and must comply with
regulations requiring that data remain within the European Union. Which two GCP
configurations should the data engineer implement? *(Choose two)*

**A)** Store all data in a multi-region bucket set to `US`
**B)** Use a BigQuery dataset configured to the `EU` multi-region
**C)** Apply an Organization Policy with `gcp.resourceLocations` constraint set to EU regions
**D)** Enable VPC Service Controls globally without region restrictions
**E)** Configure Cloud KMS in any region to manage encryption keys

**Answer: B and C**

> B ensures BigQuery data physically resides in EU. C enforces that no resources can be
> created outside EU regions at the org level — a defense-in-depth approach to data
> sovereignty.

---

### Question 2

A healthcare organization is deploying a data pipeline on GCP and must ensure HIPAA
compliance. They need to prevent authorized users from accidentally exfiltrating PHI
data to personal GCP projects. What is the most effective control?

**A)** Require all engineers to sign a data use agreement
**B)** Enable Cloud Audit Logs on all services
**C)** Implement VPC Service Controls around the project containing PHI data
**D)** Use Cloud DLP to scan all outgoing data

**Answer: C**

> VPC Service Controls create a security perimeter that prevents data from leaving the
> protected boundary even if a user has IAM permissions. Cloud DLP (D) is complementary
> but not a preventive perimeter control.

---

### Question 3

Your company requires that encryption keys for BigQuery data be rotated every 90 days
and managed internally, not by Google. Which solution meets this requirement?

**A)** Use Google-managed encryption keys (default BigQuery encryption)
**B)** Enable BigQuery column-level encryption with AEAD functions
**C)** Configure BigQuery to use Customer-Managed Encryption Keys (CMEK) via Cloud KMS
**D)** Store data in Cloud Storage with server-side encryption enabled

**Answer: C**

> CMEK via Cloud KMS gives you control over key rotation schedules and lifecycle,
> satisfying the 90-day rotation and internal management requirements. Google-managed
> keys (A) don't allow customer-controlled rotation.
