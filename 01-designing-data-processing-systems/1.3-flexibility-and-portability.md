## Key Concepts

> _Fill in your own notes here as you study._

- **Data staging**: Intermediate storage area where raw data is held before transformation.
  Typically Cloud Storage buckets used as landing zones. Enables decoupling of ingestion
  from processing and allows reprocessing from source.

- **Data cataloging**: Metadata management — organizing, tagging, and making datasets
  discoverable. Dataplex Catalog (formerly Data Catalog) is GCP's managed service.
  Supports technical metadata (schema, lineage) and business metadata (tags, descriptions).

- **Data profiling**: Automated analysis of datasets to understand structure, content,
  and quality — including value distributions, null rates, and cardinality. Dataplex
  Data Quality and Dataprep perform profiling.

- **Data discovery**: Enabling users and systems to find relevant datasets across an
  organization. Relies on a well-maintained catalog with search and tagging.

- **Data lineage**: Tracking data from source to destination through all transformations.
  Helps with impact analysis, debugging, and regulatory audits.

- **Data portability**: Designing systems so data and pipelines can move between
  environments (on-prem, multi-cloud, different GCP regions) with minimal rework.
  Open formats (Parquet, Avro, ORC) and open APIs support portability.

## Practice Questions

### Question 1

A large enterprise has data scattered across multiple BigQuery datasets, Cloud Storage
buckets, and Cloud SQL instances. Data analysts struggle to find relevant tables and
understand what data is available. Which GCP capability should a data engineer configure
to address this?

**A)** Set up BigQuery authorized views for each dataset
**B)** Configure Dataplex Catalog to scan, tag, and index assets across all sources
**C)** Create a shared spreadsheet listing all datasets and their owners
**D)** Enable BigQuery Data Transfer Service

**Answer: B**

> Dataplex Catalog automatically scans connected data sources, extracts metadata, and
> makes it searchable — directly addressing the discoverability problem. Authorized views
> (A) are for access control, not discovery.

---

### Question 2

A company ingests raw IoT sensor data into Cloud Storage before it's transformed and
loaded into BigQuery. The pipeline needs to support reprocessing historical data without
re-ingesting from the source systems. Which design pattern does this represent?

**A)** Lambda architecture
**B)** Data staging with a landing zone
**C)** Kappa architecture
**D)** Federated query pattern

**Answer: B**

> Storing raw data in Cloud Storage as a landing zone (staging area) before processing
> is a fundamental data staging pattern. It decouples ingestion from transformation and
> allows historical reprocessing without hitting source systems again.

---

### Question 3

A data governance team needs to track how a specific field in a BigQuery reporting table
was derived — from which source system, through which transformations, and which jobs
modified it. Which GCP feature provides this capability?

**A)** BigQuery INFORMATION_SCHEMA views
**B)** Cloud Audit Logs
**C)** Dataplex data lineage
**D)** Cloud Monitoring dashboards

**Answer: C**

> Dataplex captures data lineage — the end-to-end flow of data from sources through
> transformations to destinations. INFORMATION_SCHEMA (A) shows schema metadata but
> not lineage. Audit Logs (B) show who accessed data, not how it was derived.
