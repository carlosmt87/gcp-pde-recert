# Section 1: Designing Data Processing Systems

**Exam Weight: ~25%**

This is one of the highest-weighted sections. It covers the architectural decisions a
data engineer must make when designing secure, reliable, and flexible data systems on
Google Cloud.

## Subsections

| Subsection | Topic | File |
|------------|-------|------|
| 1.1 | [Designing for security and compliance](./1.1-security-and-compliance.md) | Security, data sovereignty, regulatory compliance |
| 1.2 | [Designing for reliability and fidelity](./1.2-reliability-and-fidelity.md) | Data preparation, cleaning, validation |
| 1.3 | [Designing for flexibility and portability](./1.3-flexibility-and-portability.md) | Data staging, cataloging, profiling, discovery |

## Key Themes

- **Security by design**: IAM, VPC Service Controls, CMEK, data residency
- **Data quality**: Validation at ingestion and transformation layers
- **Governance**: Cataloging, data lineage, and discovery
- **AI-assisted engineering**: Using LLMs to generate SQL/transformations
